# PointAsk Privacy Boundaries

## Data PointAsk may process

Only as part of an explicit user flow, PointAsk may process:

- Source text the user actively selects.
- The paragraph containing that source text, as context.
- The local follow-up question entered by the user.
- The prompt generated by PointAsk.
- Answer text the user actively selects and chooses to attach.
- A uniquely matched, completed Workspace answer after the user explicitly clicks “继续追问”, or after the user completes a partial attachment while leaving that staged round unselected. It is stored only in `chrome.storage.local`; attached copies are removed immediately and skipped copies expire after at most 30 days or are removed with thread/all-data deletion.
- Relevant ChatGPT page URLs used to restore a thread or navigate between its source and target.

Application data is stored locally by default with `chrome.storage.local`.

## Data PointAsk does not process

PointAsk does not process or collect:

- ChatGPT cookies.
- Passwords.
- Login or session tokens.
- Complete browsing history.
- Other unselected page text.
- Complete AI answers unless the user explicitly selects them, clicks “继续追问”, or completes partial attachment that locally retains an unselected staged round for at most 30 days.
- Automatically generated background tasks.
- Data stored on an external server.

PointAsk does not use a model API, a private ChatGPT interface, account authentication access, hidden-tab automation, or automatic message submission and answer extraction. It never stages an answer during page load, timers, restoration, or background activity.
